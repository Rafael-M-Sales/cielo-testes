Sempre que você (o agente) concluir uma tarefa de codificação, refatoração ou configuração que resulte em alterações no sistema de arquivos:
1. Realize o `git add .`.
2. Realize o `git commit` com uma mensagem clara e descritiva seguindo o padrão Conventional Commits (ex: `feat:`, `fix:`, `docs:`).
3. Realize o `git push` para o repositório remoto.
Isso deve ser feito de forma proativa, sem necessidade de pedido explícito a cada vez, após garantir que a alteração está correta.
